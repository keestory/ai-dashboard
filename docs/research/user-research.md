# AI Dashboard 사용자 리서치 보고서

## 1. 타겟 사용자 페르소나

### 페르소나 1: 스타트업 운영 매니저 "김민수"
- **나이**: 32세
- **직책**: 운영 매니저 (시리즈 A 스타트업)
- **팀 규모**: 3명
- **기술 수준**: 중급 (Excel 능숙, SQL 기초)

#### 현재 상황
- 매주 여러 부서에서 Excel 파일로 데이터 수집
- 수동으로 피벗테이블 만들어 분석
- 월간 보고서 작성에 2일 소요
- 데이터 분석가 채용 예산 없음

#### 페인 포인트
1. 데이터 합치고 정리하는데 시간 많이 소요
2. 이상치나 트렌드 발견하기 어려움
3. 경영진에게 인사이트 설명하기 어려움
4. "그래서 뭘 해야 하는데?" 질문에 답하기 어려움

#### 니즈
- 파일 올리면 자동으로 분석
- 비전문가도 이해할 수 있는 시각화
- 액션 아이템 제안
- 빠른 리포트 생성

#### 기대하는 가치
- 분석 시간 80% 단축
- 데이터 기반 의사결정 가능
- 경영진 보고 품질 향상

- **직책**: 대표이사 (제조업 50명 규모)
- **기술 수준**: 초급 (Excel 기본)

#### 현재 상황
- 회계팀에서 월별 매출 Excel 받음
- 숫자만 보고 감으로 의사결정
- BI 도구 도입 시도했으나 복잡해서 포기
- IT 담당자 부재

#### 페인 포인트
1. 데이터는 있는데 활용을 못함
2. 직원들에게 분석 요청하면 시간 오래 걸림
3. 시장 변화 놓치는 경우 많음
4. 경쟁사 대비 데이터 활용 뒤처짐

#### 니즈
- 복잡한 설정 없이 바로 사용
- 한눈에 파악되는 대시보드
- 문제점 자동 알림
- 모바일로 언제든 확인

#### 기대하는 가치
- 사업 현황 실시간 파악
- 빠른 의사결정
- 비용 절감 기회 발견

- 고부가가치 작업에 집중
- 전문성 어필

## 2. 사용자 여정 맵 (User Journey Map)

### 단계 1: 인지 (Awareness)
- **상황**: "Excel 분석 너무 오래 걸려..."
- **행동**: 구글 검색 "엑셀 자동 분석 도구"
- **감정**: 답답함, 기대
- **터치포인트**: 검색 광고, 블로그 리뷰

### 단계 2: 고려 (Consideration)
- **상황**: 여러 도구 비교
- **행동**: 무료 체험, 데모 영상 시청
- **감정**: 비교, 평가
- **터치포인트**: 랜딩 페이지, 데모 영상, 가격표

### 단계 3: 가입 (Signup)
- **상황**: 무료 시작 결정
- **행동**: 이메일로 가입
- **감정**: 기대, 약간의 불안
- **터치포인트**: 가입 폼, 온보딩 이메일

### 단계 4: 첫 사용 (First Use)
- **상황**: 첫 파일 업로드
- **행동**: 샘플 파일 또는 실제 파일 업로드
- **감정**: 호기심, 긴장
- **터치포인트**: 업로드 UI, 진행 상태, 결과 화면
- **핵심**: 3분 내 "와!" 모먼트 제공

### 단계 5: 가치 발견 (Aha Moment)
- **상황**: AI가 인사이트 제공
- **행동**: 인사이트 확인, 차트 탐색
- **감정**: 놀라움, 만족
- **터치포인트**: 인사이트 카드, 대시보드
- **핵심**: "내가 몰랐던 걸 AI가 찾아줬다!"

### 단계 6: 습관화 (Habit)
- **상황**: 정기적 사용
- **행동**: 주간 데이터 업로드, 리포트 생성
- **감정**: 의존, 신뢰
- **터치포인트**: 알림, 예약 분석

### 단계 7: 확장 (Expansion)
- **상황**: 팀원 추가, 기능 확대
- **행동**: 유료 전환, 팀 초대
- **감정**: 가치 인정, 투자 정당화
- **터치포인트**: 업그레이드 제안, 팀 기능

### 단계 8: 옹호 (Advocacy)
- **상황**: 추천
- **행동**: 동료에게 추천, 리뷰 작성
- **감정**: 자부심, 공유 욕구
- **터치포인트**: 추천 프로그램, 리뷰 요청

## 3. 핵심 사용자 니즈

### 3.1 기능적 니즈
| 우선순위 | 니즈 | 설명 |
|---------|------|------|
| P0 | 쉬운 업로드 | 드래그앤드롭, 다양한 파일 형식 |
| P0 | 자동 분석 | 업로드만 하면 분석 시작 |
| P0 | 시각화 대시보드 | 직관적인 차트와 그래프 |
| P1 | AI 인사이트 | 패턴, 이상치, 트렌드 자동 발견 |
| P1 | 액션 추천 | "이렇게 하세요" 제안 |
| P1 | 리포트 내보내기 | PDF, Excel 다운로드 |
| P2 | 알림 | 이상 감지 시 알림 |
| P2 | 팀 협업 | 대시보드 공유, 코멘트 |
| P3 | API | 자동화 연동 |

### 3.2 감정적 니즈
1. **자신감**: "나도 데이터 분석 할 수 있어"
2. **안심**: "놓치는 게 없겠지"
3. **효율감**: "시간을 아꼈어"
4. **전문성**: "프로처럼 보여"

### 3.3 사회적 니즈
1. **인정**: 경영진/팀에게 인정받기
2. **공유**: 성과 보여주기
3. **협업**: 팀과 같은 데이터 보기

5. 담당자에게 공유

## 5. UX 핵심 원칙

### 5.1 온보딩
- 가입 후 30초 내 첫 업로드
- 샘플 데이터로 즉시 체험 가능
- 프로그레시브 디스클로저 (단계별 기능 공개)

### 5.2 분석 경험
- 업로드 → 분석 → 인사이트: 1분 이내
- "왜 이 인사이트인가" 설명 제공
- 자연어로 추가 질문 가능

### 5.3 시각화
- 모바일 퍼스트 디자인
- 탭 한번에 상세 정보
- 다크 모드 지원

### 5.4 액션
- 추천 액션 명확하고 실행 가능
- 원클릭 리포트 생성
- 원클릭 공유

## 6. 성공 지표

### 6.1 활성화 지표
- 가입 → 첫 분석 완료율: 목표 60%
- 첫 주 재방문율: 목표 40%
- Time to Value: 목표 3분 이내

### 6.2 참여 지표
- 주간 활성 사용자 (WAU)
- 사용자당 월 분석 횟수
- 대시보드 체류 시간

### 6.3 수익 지표
- 무료 → 유료 전환율: 목표 5%
- 월간 반복 매출 (MRR)
- 사용자당 평균 수익 (ARPU)

### 6.4 만족 지표
- NPS: 목표 50+
- 추천 전환율
- 해지율: 목표 5% 이하

## 7. 결론

### 핵심 인사이트
1. **단순함이 핵심**: 기술 비전문가도 3분 내 가치 체험
2. **인사이트 넘어 액션**: "그래서 뭘 해야 해?"에 답하기
3. **모바일 필수**: 언제 어디서나 확인 니즈
4. **신뢰 구축**: AI 결과에 대한 설명 필요

### 우선 개발 기능
1. 드래그앤드롭 파일 업로드
2. 자동 데이터 분석 및 시각화
3. AI 인사이트 카드
4. 액션 추천
5. PDF 리포트 내보내기
6. 모바일 대시보드

### 차별화 경험
- "업로드 후 1분 내 인사이트"
- "데이터 전문가 없이도 프로 수준 분석"
- "분석에서 액션까지 원스톱"

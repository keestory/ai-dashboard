## 버그 리포트 템플릿

### docs/qa/bug-report-template.md

```markdown
# 버그 리포트

## 기본 정보
- **ID**: BUG-XXX
- **제목**: [간단한 설명]
- **심각도**: Critical / High / Medium / Low
- **우선순위**: P0 / P1 / P2 / P3
- **발견자**: [이름]
- **발견일**: YYYY-MM-DD

## 환경
- **플랫폼**: Web / iOS / Android
- **OS 버전**: [예: iOS 17.0]
- **앱 버전**: [예: 1.0.0]
- **브라우저**: [예: Chrome 120]
- **디바이스**: [예: iPhone 15 Pro]

## 재현 단계
1. [첫 번째 단계]
2. [두 번째 단계]
3. [세 번째 단계]

## 예상 결과
[이렇게 동작해야 함]

## 실제 결과
[이렇게 동작함]

## 스크린샷/영상
[첨부]

## 추가 정보
- 재현율: 항상 / 가끔 / 드물게
- 관련 로그: [첨부]
- 관련 이슈: #XXX
```

## 심각도 기준

| 심각도 | 설명 | 예시 |
|--------|------|------|
| Critical | 서비스 불가 | 앱 크래시, 로그인 불가 |
| High | 핵심 기능 장애 | 저장 안 됨, 데이터 손실 |
| Medium | 기능 일부 문제 | UI 깨짐, 느린 응답 |
| Low | 사소한 문제 | 오타, 미세한 정렬 |

## 주의사항

- 출시 전 모든 Critical/High 이슈 해결 필수
- 테스트는 실제 환경과 유��하게 (Production 데이터)
- 에지 케이스 충분히 테스트
- 재현 단계는 상세하게 작성
